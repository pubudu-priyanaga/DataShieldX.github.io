Research Domain

Literature Survey

Our research builds upon existing DLP solutions, which primarily rely on content inspection and rule-based blocking. We identified that traditional DLP tools face limitations in tackling modern threats, which require more adaptive and multi-layered protection methods. Comparative analysis of commercial DLP tools like Symantec (Broadcom), Forcepoint, Trellix, and Microsoft Purview revealed gaps in detection capabilities and performance overhead concerns.

Open-source alternatives such as MyDLP provide basic monitoring features but lack comprehensive protection against sophisticated exfiltration attempts and insider threats. Our literature review also covered privacy-preserving data inspection techniques, user behavior analytics for insider threat detection, and advancements in deep learning for steganalysis.

Research Gap

Despite advancements in DLP technologies, several critical gaps remain:

  • Existing DLP solutions lack integration of advanced machine learning for real-time detection
  • Limited capabilities in detecting covert data exfiltration channels such as steganography
  • Insufficient focus on user behavior analysis and risk scoring
  • Inadequate mechanisms for secure logging of sensitive data breaches
  • Poor multi-platform support across different operating systems

Research Problem

Organizations face significant risks through data exfiltration combined with insider threats and compliance violations, especially under regulatory standards such as PCI DSS, GDPR, and HIPAA. Existing solutions fail to provide comprehensive protection against sophisticated exfiltration attempts while maintaining operational efficiency.

Research Objectives

  1. Develop an intelligent data classification system capable of detecting sensitive information across multiple regulatory frameworks
  2. Create a dynamic risk scoring framework that assigns risk levels based on user violations and behavioral patterns
  3. Implement an endpoint security module for webcam-based login verification and phishing detection
  4. Design a deep learning-based steganalysis engine to detect data exfiltration via steganographic images
  5. Integrate homomorphic encryption for secure logging of sensitive data breaches

Methodology

Our research methodology encompasses:

Data Collection & Preparation

Using datasets such as ALASKA2, BOSSbase, and BOWS2 for steganalysis model training, along with synthetic and real samples of sensitive data for classification model training.

Model Development

Implementing machine learning models including ALBERT for PII detection, CNN architectures for steganalysis, and ensemble methods for phishing URL detection.

System Integration

Combining multiple components into a unified DLP system with homomorphic encryption for secure logging and a risk scoring framework for insider threat detection.

Performance Evaluation

Conducting comprehensive testing to measure accuracy, precision, recall, and F1-scores across all system components.
